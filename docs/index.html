Getting Started

Up and running in minutes

Install Prerequisites

Rust nightly toolchain and cmake. Optional: ffmpeg for voice transcription.

rustup toolchain install nightly
rustup override set nightly
sudo apt install cmake

Build

Clone and build with all channels, or select only what you need.

git clone https://github.com/oxicrab/oxicrab
cd oxicrab
cargo build --release

Configure

Create your config file with API keys and channel tokens.

mkdir -p ~/.oxicrab
cp config.example.json ~/.oxicrab/config.json
# Edit with your API keys

Run

Start the gateway for all channels, or use CLI mode for quick interactions.

# Multi-channel daemon
./target/release/oxicrab gateway

# CLI mode
./target/release/oxicrab agent -m "Hello!"

# First-time setup
./target/release/oxicrab onboard
Configuration

Simple JSON configuration

One config file controls everything. camelCase in JSON, snake_case in Rust.

{
  "agents": {
    "defaults": {
      "model": "claude-sonnet-4-5-20250929",
      "maxTokens": 8192,
      "maxToolIterations": 20,
      "memory": {
        "embeddingsEnabled": false,
        "hybridWeight": 0.5
      },
      "compaction": {
        "enabled": true,
        "thresholdTokens": 40000
      }
    }
  },
  "providers": {
    "anthropic": { "apiKey": "sk-ant-..." }
  },
  "channels": {
    "telegram": {
      "enabled": true,
      "token": "123456:ABC...",
      "allowFrom": ["your_user_id"]
    }
  }
}

Architecture

Clean, modular, extensible

Built on three core traits with a middleware pipeline for tool execution.

Channel → MessageBus → AgentLoop → LLM Provider
LLM call → parallel tool execution → append to conversation → repeat

trait Tool

name(), description(), parameters() (JSON Schema), execute(). Optional: cacheable(). Registered in ToolRegistry with middleware pipeline.

trait BaseChannel

start(), stop(), send(). Optional: send_typing(), edit_message(), delete_message(). Auto-reconnect with exponential backoff.

trait LLMProvider

chat(ChatRequest) → LLMResponse. Default chat_with_retry() with exponential backoff. warmup() pre-warms HTTP connections.

ToolMiddleware

before_execute() can short-circuit, after_execute() can modify results. Built-in: CacheMiddleware, TruncationMiddleware, LoggingMiddleware.

ToolRegistry

Central execution engine with panic isolation via tokio::task::spawn, timeouts, LRU caching (128 entries, 5min TTL), and parallel execution.

ProviderFactory

Automatic provider selection by model name. OAuth-first for Anthropic, then API key strategy with OpenAI-compatible matching.
